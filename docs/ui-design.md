> 文档版本: 1.0 | 生成日期: 2026-05-13

## 1. 项目概览

本项目是一款以 18 世纪大航海时代为背景的回合制策略游戏，使用 UrhoX 引擎 + Lua 开发。UI 系统基于 `urhox-libs/UI`（Yoga Flexbox 布局 + NanoVG 渲染），辅以自定义 NanoVG Widget 实现六边形地图。

### 技术栈

| 层级 | 技术 |
|------|------|
| 布局引擎 | Yoga Flexbox |
| 渲染后端 | NanoVG |
| UI 组件库 | urhox-libs/UI（Panel, Label, Button, ProgressBar, ScrollView 等） |
| 自定义渲染 | WorldMapWidget（继承 Widget 基类，NanoVG 直接绘制六边形地图） |

| 点击底栏导航 | S1/S2 | S3 | `GoS3(moduleId)` |
| 点击浮动卡片"查看详情" | S2 | S3 | `GoS3("build")` |

| 点击"返回" | S3/S4 | S1 | `GoBack()` |
| 点击✕关闭 | S2 | S1 | `GoS1()` |

### 4.4 WorldMap（六边形世界地图）

**文件**: `ui/WorldMap.lua` | **类型**: 自定义 NanoVG Widget

基于 `Widget:Extend("WorldMapWidget")` 实现的自定义控件，使用 NanoVG 直接绘制 Pointy-top 六边形网格地图。

#### 六边形参数

| 参数 | 公式/值 | 说明 |
|------|---------|------|
| 边长 | `HEX_EDGE = 16px` | 基础尺寸 |
| 水平宽度 | `√3 × 16 ≈ 27.7px` | 单个瓦片宽 |
| 垂直高度 | `2 × 16 = 32px` | 单个瓦片高 |
| 列间距 | `= HEX_W` | 水平步进 |
| 行间距 | `= HEX_H × 0.75 = 24px` | 垂直步进（3/4 高度） |
| 奇数行偏移 | `+ COL_STEP × 0.5` | 奇数行右移半格 |

#### 地图范围

- 22 列 × 12 行，覆盖欧洲、大西洋、北美、加勒比、南美、非洲
- 10 种瓦片类型：深海、浅海、欧洲城市/港口、殖民地陆地/港口、大西洋、非洲、美洲、未探索区
- 6 条贸易航线（贝塞尔曲线绘制）
- 底部左侧图例框

#### 交互支持

| 操作 | 实现 | 参数 |
|------|------|------|
| 平移 | 拖拽（PointerDown/Move/Up） | 无限制 |
| 缩放 | 滚轮（OnWheel） | 0.7x ~ 2.5x |
| 点击 | 像素→hex 坐标转换（cube rounding） | 检测港口→回调 |

#### 渲染层次

1. 海洋渐变背景（`nvgLinearGradient`）
2. 所有瓦片（填充 + 描边，裁剪视口外瓦片）
3. 港口标记（⚓ emoji）
4. 选中港口高亮（金色外圈）
5. 瓦片名称（zoom ≥ 1.0 时显示）
6. 贸易航线（弧线 + 中点标记）
7. 图例框（左下角）

## 5. 数据流与回调

### 5.1 回调链路

```
用户操作 → 组件回调 → main.lua 处理函数 → 状态更新 → RefreshUI()

RefreshUI() 调用 CreateUI() 重建整个 UI 树 → UI.SetRoot(root)
```

### 5.2 主要回调

| 来源 | 回调 | 处理逻辑 |
|------|------|---------|
| WorldMap | `onPortClick(portId)` | 查找殖民地 → GoS2 |
| BottomBar | `onNav(navId)` | GoS3(navId) |
| BottomBar | `onEndTurn()` | GameEngine.EndTurn → GoS4 |
| ColonyPanel | `onColonyAction(action, id)` | GameEngine.ColonyAction → 扣资源 → Refresh |
| TechPanel | `onResearchTech(techId)` | GameEngine.StartResearch → Refresh |
| TradePanel | `onTradeAction(action)` | 新建/升级路线 → Refresh |
| BattlePanel | `onBattleAction(action)` | 追击/撤退 → Refresh |
| GamePanel | `onClose()` | GoBack() |

### 5.3 状态管理

全局状态存储在 `state_`（GameState 实例），所有 UI 组件通过 props 接收状态的引用，不直接修改状态。状态修改统一通过 `main.lua` 中的回调处理函数完成，修改后调用 `RefreshUI()` 重建 UI。

## 7. 设计模式总结

### 面板容器模式

所有 S3/S4 面板遵循统一结构: `Header(深木色) → SubTabs(可选) → ScrollView(羊皮纸)`，由 GamePanel 封装。

### 区块标题模式

金色竖条 (3×12px) + 粗体标题文字，用于面板内区块分隔。

### 状态标签模式

圆角矩形 Badge，按状态着色（绿/橙/红），用于风险等级、外交状态、科技状态等。

### 进度条模式

`标签(固定宽) | 彩色填充条(固定宽) | 数值(右对齐)`，用于属性值、关系值、控制度等。

### 双栏布局模式

`左侧(flexGrow=1) | 1px分隔线 | 右侧(固定宽)`，用于年度报告等信息对比展示。
