## 1. 设计语言基础

### 1.1 视觉哲学

> **「一张活着的航海图」**

整个游戏界面应像一张摊开在船长桌上的古典航海图：地图是主角，UI 是压在地图边缘的导航仪器。每个面板像用黑墨水和金箔绘制在羊皮纸上，每个按钮像经过海水与时光侵蚀的铜质舰船零件。

- 地图区域：手绘风格，细腻的水纹、山脉阴影线、海怪装饰角落
- UI 框架：深色木纹边框 + 铜/金铆钉装饰 + 羊皮纸底面
- 数据呈现：黑色羽毛笔手写风格，数字用衬线体
- 动效：缓慢展开（如展卷轴）而非快速弹入

### 1.5 纹理与装饰元素

#### 必用纹理层

| 纹理 | 用途 | 透明度 |
|------|------|------|
| 羊皮纸噪点纹理（PNG overlay）| 所有面板底色叠加 | 20-35% |
| 木纹纹理 | 顶栏/底栏/面板边框 | 50% |
| 旧纸折痕（轻微）| 主背景 | 15% |

#### 装饰元素库

- **铆钉**: 四角装饰，圆形铜色，出现在面板角落（6×6px）
- **分隔线**: 水平线用粗细不均的「墨线」而非直线，两端有小装饰花
- **角花**: 卷草纹或锚/帆船剪影，用于面板四角装饰框
- **红色蜡封**: 重要操作确认弹窗的装饰元素
- **罗盘玫瑰**: 地图右下角固定装饰
- **海怪/鲸鱼**: 深海区域的装饰（不影响交互）
- **航线虚线**: 贸易路线用古典虚线+小帆船图标标注

## 2. 布局架构

### 2.1 基准尺寸

| 设备 | 分辨率 | 设计适配策略 |
|------|--------|-----------|
| iPhone 14（基准）| 844 × 390px | 1:1 像素设计 |
| iPhone SE 横屏 | 667 × 375px | 收窄右面板 |
| iPhone 14 Plus | 932 × 430px | 地图区域扩展 |
| iPad 横屏（兼容）| 1024 × 768px | 右面板常驻显示 |

安全区域（刘海/圆角屏）：左右各 44px，上下各 8px 的 safe area padding。

---

### 2.2 三大区域划分

```
┌──────────────────────────────────────────────────────────────────────┐
│                    ▲ TOP BAR — 40px 固定高度                         │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│                  ▼ MAIN AREA — 300px 可变高度                        │
│                                                                      │
│   ┌─────────────────────────────────────────┐ ┌───────────────────┐ │
│   │                                         │ │                   │ │
│   │          MAP ZONE                       │ │   SIDE PANEL      │ │
│   │          （默认：全宽 844px）            │ │   （选中后滑入）  │ │
│   │          （选中后：约 524px）            │ │   （320px 宽）    │ │
│   │                                         │ │                   │ │
│   └─────────────────────────────────────────┘ └───────────────────┘ │
│                                                                      │
├──────────────────────────────────────────────────────────────────────┤
│                    ▲ BOTTOM BAR — 50px 固定高度                      │
└──────────────────────────────────────────────────────────────────────┘
```

| 区域 | 高度 | 行为 |
|------|------|------|
| TOP BAR | 40px | 固定，永不消失 |
| MAIN AREA | 300px | 地图可缩放/平移 |
| BOTTOM BAR | 50px | 固定，内容随状态变化 |

---

### 2.3 TOP BAR 内部结构

```
[家族徽章 30px] [💰XXXX] [🔬XXX] [⚙️XXX] [👑XX] [⚓XX] [🌍XX]  [1643年] [AP:⬡⬡⬡⬡⬡] [☰]
 ← 左区 →       ←────────────── 中区（数值栏）──────────────→  ←── 右区 ───────────────→
```

| 子区 | 内容 | 宽度 |
|------|------|------|
| 左区 | 家族徽章（点击进家族管理）| 40px |
| 中区 | 6个数值图标+数字（点击任意数值展开详情）| ~600px |
| 右区 | 当前年份 + AP沙漏 + 汉堡菜单 | ~160px |

**数值项单元（中区）：**

```
┌───────────┐
│ [图标]    │  ← 20×20px 蚀刻图标
│ 2,480     │  ← 数字，Libre Baskerville 14px, --ink-0
│ +320/年   │  ← 产出速率，10px，--status-ok 绿色（正）/ --status-war 红色（负）
└───────────┘
宽度约 80px/个，间距 6px
```

数值不足（如财富 < 0）时：数字变为 `--status-war`，图标轻微抖动（pulse 动画）。

### 2.4 BOTTOM BAR 内部结构

底栏随当前状态呈现三种形态：

**形态 A：默认（无选中）**

```
[🏗️建设] [📦贸易] [🤝外交] [⚓海军] [👨‍👩‍👧家族] [🔬科技]              [结束回合 →]
 ←───────────────── 功能导航按钮组 ─────────────────────→              ←── 主行动 ──→
```

**形态 B：地块已选中**

```
[✕关闭] [地块名称（截断）]  [🏗️建造] [🏴占领] [🔍探险] ... [结束回合 →]
 ← 取消选中 →  ← 地块信息 →  ←── 当前地块可用快捷操作（动态生成）──→  ← 主行动 →
```

**形态 C：功能模块打开中**

```
[← 返回]  [当前模块名]                                       [确认/执行]
```

## 3. 屏幕状态系统

### 3.1 五种主状态

```
S0: 启动/加载         ─── 全屏航海图展开动画
S1: 主地图（空闲）    ─── 默认游戏状态
S2: 地块已选中        ─── 右侧面板滑入
S3: 功能模块打开      ─── Bottom Sheet 从下展开（半高~65%）
S4: 全屏弹窗          ─── 事件/年度报告/胜利画面
```

### 3.2 状态转换规则

```
S1 ──[单击地块]──────────────────→ S2
S1 ──[点击底栏功能按钮]──────────→ S3
S2 ──[单击地图空白处 / ✕]────────→ S1
S2 ──[点击底栏快捷操作]──────────→ S3（叠加在 S2 上）
S3 ──[← 返回 / 点击遮罩]────────→ 返回前一状态
S3 ──[确认/执行]────────────────→ 返回 S1/S2 + 动效反馈
S4 ──[关闭/确认]────────────────→ S1
任意状态 ──[新事件触发]──────────→ S4（强制置顶）
```

### 4.2 进度条组件

用于控制度、科技进度、声望/数值里程碑。

| 声望里程碑 | `--gold-bright` |
| 抵抗运动强度 | `--status-war` |

### 4.3 数值变化提示（Toast）

位置：屏幕右上角，TOP BAR 下方叠加。每条 Toast 高度 28px，最多堆叠 4 条。

### 5.1 六边形色彩编码系统

每格六边形边长约 24px（844px 宽度内容纳约 22 列 × 12 行 = ~180 格）。

| 地块类型 | 填充颜色 | 边框颜色 | 特殊标记 |
|---------|---------|---------|---------|

| 欧洲港口 | `#A0B8C8`（海灰蓝）| `#2E6B8A` | 锚图标 |
| 殖民荒地 | `#7A9858`（野绿）| `#5A7840` | — |
| 已殖民（玩家）| 玩家家族色 40%透明 | 家族色 | 旗帜图标 |
| 已殖民（AI）| AI家族色 40%透明 | 家族色 | 旗帜图标（AI色）|
| 资源地块（叠加）| 地块底色 | — | 资源小图标居中 |
| 浅海 | `#5AA0C0`（浅蓝）| `#3A80A0` | 小浪纹纹理 |
| 深海 | `#1A4A7A`（深海蓝）| `#0F2A50` | 水流纹理 |
| 大西洋航线 | `#0F2A4A` | `#0A1E38` | 偶有海怪装饰 |

#### 控制度视觉映射

控制度 0-100% 通过透明度表达：

```
控制度 0-29%:  家族色透明度 15%（几乎不可见）
控制度 30-49%: 家族色透明度 30%
控制度 50-69%: 家族色透明度 50%
控制度 70-89%: 家族色透明度 70%
控制度 90-100%: 家族色透明度 90% + 边框加粗 2px
```

#### 战争/封锁状态

```
战争格:    红色虚线边框（animated dash，速度慢）
封锁港口:  锚图标变红色 + 小锁图标叠加
```

| 规则 | 标准值 |
|------|--------|
| 最小触控目标 | 44 × 44 px |
| 地图六边形边长 | 24px（可触控面积约 50px²，足够）|
| 底栏按钮宽度 | 60px（包含图标+文字）|
| 按钮间距 | 最小 4px（防误触）|
| 滚动惯性 | 标准 iOS/Android 弹性滚动 |
| 长按触发时长 | 不使用长按（见 §3 设计决策）|

### 7.2 手势系统

```
单击地块     → 选中（S2）
单击空白     → 取消选中（S2→S1）
单击底栏按钮  → 打开模块（S3）
单击操作卡片  → 执行确认流程
双击地图     → 聚焦并放大到该格（缩放×1.5）
双指缩放     → 地图缩放
单指滑动     → 地图平移
下拉 Sheet   → 关闭 Sheet（超过30%拖拽距离触发）
```

### 7.3 操作确认流程

**高代价操作**（AP ≥ 2 或 财富 ≥ 1000）需二次确认：

```
第一次点击 → 按钮变为「确认状态」（颜色变深，文字变「确认操作？」）
第二次点击（2秒内）→ 执行
超时未确认 → 回复原状态（无操作）
取消 → 点击其他区域
```

**低代价操作**（AP = 1，财富 < 500）：单次点击直接执行。

**危险操作**（宣战/放弃殖民地）：全屏确认弹窗，要求滑动确认条（防误触）。

---

### 7.4 数值变化反馈

执行操作后，TOP BAR 中对应数值图标触发：

```
正向变化: 图标向上弹跳（translateY -4px，200ms），数字从旧值到新值滚动
负向变化: 图标向右抖动（shake，150ms），数字红色闪烁后归零
数值不足: 图标持续 pulse（发光，循环 1s）+ 提示 Toast
```

## 8. 动效设计

### 8.1 动效原则

> 所有动效模拟「纸张/卷轴/铜件」的物理特性：展开、折叠、沉降——而非数字弹跳或闪光。

| 动效类型 | 时长 | 缓动 | 说明 |
|---------|------|------|------|
| 面板滑入（S2）| 280ms | ease-out | 右侧面板从右滑入 |
| 面板滑出 | 200ms | ease-in | |
| Sheet 展开 | 320ms | spring（0.7,0.8）| 从底部展开，轻微回弹 |
| Sheet 收起 | 220ms | ease-in | |
| 全屏弹窗进入 | 300ms | ease-out | scale 0.88→1.0 + fade in |
| 全屏弹窗退出 | 200ms | ease-in | fade out |
| 地图选中高亮 | 120ms | ease-out | 六边形边框亮起 |
| 数值滚动 | 400ms | ease-out | 数字滚动到新值 |
| Toast 进入 | 200ms | ease-out | 从右滑入 |
| Toast 消失 | 500ms | ease-in | 淡出 |
| 页面切换 | 300ms | ease-in-out | 水平 swipe |

### 8.2 特殊动效

#### 回合结束动画（1.5秒）

```
1. 底栏「结束回合」按钮向右扫出（100ms）
2. 整个地图覆盖一层半透明沙漏图案（淡入 200ms）
3. 年份数字从旧年滚动到新年（600ms，中央大字）
4. 沙漏图案淡出（200ms）
5. TOP BAR 各数值依次更新（每个间隔 80ms，从左到右）
6. 若有年度报告 → 弹出
```

#### 贸易路线建立动画（1秒）

```
新路线从起点「生长」至终点，以虚线逐段出现（path animation），
终点出现小帆船图标，同时 Toast 显示路线收益。
```

#### 殖民地控制度突破动画

```
控制度达到 50% / 70% / 90% 节点时：
地块六边形短暂放大（scale 1.15）→ 恢复（弹性），
同时边框颜色从浅到深过渡，
Sound：硬币落地声（可选）
```

#### 科技解锁动画（1.2秒）

```
科技节点从「虚线→实心」过渡（300ms），
金色光芒从节点向相连路径蔓延（600ms），
全屏短暂叠加「科技效果文字」卡片（300ms 渐入 → 2s 停留 → 200ms 淡出）
```

#### 历史事件卡片进入（0.5秒）

```
背景遮罩先淡入（200ms）
弹窗从下方轻微浮起（translateY 20→0，同步 fade in，300ms）
蜡封图案延迟 200ms 旋转出现（0→1 scale，100ms）
```

## 9. 无障碍与适配

### 9.1 分辨率适配

| 设备 | 调整策略 |
|------|---------|
| 宽度 < 667px（小手机）| 地图缩小至 0.9x，右面板改为全屏遮罩 |
| 宽度 667-844px（主流）| 标准设计 |
| 宽度 844-932px | 地图区扩展，右面板宽度增至 360px |
| 宽度 > 1024px（平板）| 右面板常驻（不滑入），地图满宽 |

### 9.2 字号适配

系统字体放大（用户可在设置中调节）：

```
全局字号 = 基础字号 × 用户设置系数（0.85 / 1.0 / 1.15 / 1.30）
最小可接受字号: 9px（地图标注保持 9px 不缩放，防止地图混乱）
```

### 9.3 色盲适配

颜色绝不是唯一区分方式：
- 家族势力：颜色 + 家族图案纹路
- 资源类型：颜色 + 图标
- 进度条状态：颜色 + 数字百分比
- 威胁/正常：颜色 + 图标（⚠️ / ✓）

提供「色盲模式」选项，切换为高对比度 + 纹理区分方案。

### 9.4 触控辅助

- 「大按钮模式」：所有触控目标 ×1.3 尺寸（减少操作精度要求）
- 操作确认时长可在设置中延长（默认 2s → 最长 5s）
- 支持键盘/外接手柄（平板场景）
